- Next.js 14
- React 18
- TypeScript

- lucide-react

- `/orders/new` — создание нового заказа
- `/orders/[id]` — карточка заказа
- `/clients` — список клиентов

## Главные файлы

- `src/data/mock-data.ts` — единый стартовый набор mock-данных.
- `src/types/crm.ts` — основные CRM-типы.
- `src/types/order.ts` — типы формы нового заказа.
- `src/lib/crm-utils.ts` — общие функции поиска, связей, форматирования и расчетов.
- `src/lib/storage.ts` — слой localStorage для заказов, клиентов, платежей, производства и установки.
- `src/lib/order/storage.ts` — создание заказа из формы и сохранение связанных сущностей.
- `src/lib/order/validation.ts` — валидация нового заказа.
- `src/lib/order/calculations.ts` — расчеты сумм заказа.
- `tests/order-calculations.test.ts` — тесты расчетов и валидации.

## Единая структура данных

Основные сущности связаны через id:

- клиент: `client.id`
- заказ: `order.clientId`
- платеж: `payment.orderId`, `payment.clientId`
- производство: `productionTask.orderId`
- установка: `installationTask.orderId`
- документ: `document.orderId`, `document.clientId`

Один и тот же заказ должен отображаться в списке заказов, карточке заказа, карточке клиента, финансах, производстве, установке и документах.

## Сохранение нового заказа

На странице `/orders/new` кнопка `Создать заказ` должна:

1. Проверить обязательные поля.
2. Найти клиента по телефону или создать нового.
3. Сгенерировать номер заказа вида `ЗК-2026-0143`.
4. Создать заказ в localStorage.
5. Создать позиции комплектации и услуги внутри заказа.
6. Если есть предоплата, создать платеж.
7. Создать задачу производства со статусом `Ожидает макет`.
8. Создать задачу установки со статусом `Не назначена`.
9. Перенаправить пользователя на `/orders/[id]`.

Новый заказ должен появляться в:

- `/orders`
- `/orders/[id]`
- `/clients`
- `/clients/[id]`
- `/finance`
- `/production`
- `/installation`

## Черновик заказа

На `/orders/new` есть черновик:

- `Сохранить как черновик` сохраняет форму в localStorage.
- При повторном открытии формы CRM предлагает восстановить или удалить черновик.
- Есть кнопка `Очистить форму`.

## Сброс демо-данных

В `/settings`, вкладка `Общие`, есть кнопка:

```text
Сбросить демо-данные
```

Она очищает localStorage и возвращает CRM к исходным mock-данным.
